# LuoguSP 设计文档:IDE 模式「一键测试样例」

- 日期:2026-07-22
- 状态:已获 owner 审核通过(经三轮交互效果图迭代定稿)
- 分支:`feature/ide-batch-sample-test`

## 1. 目标

在洛谷题目页的 IDE 模式中新增「一键测试样例」按钮(位于原生「自测」按钮左侧)。点击后逐组驱动洛谷现有的在线自测能力,把题目全部输入样例依次跑一遍,并与期望输出对照,以「每组样例一行、可折叠展开、手风琴式(最多展开一组)」的面板呈现结果。

## 2. 交互设计(定稿)

### 2.1 入口与面板形态

- 「一键测试样例」按钮插入在原生「自测」按钮左侧,通过**克隆原生按钮元素**生成(继承洛谷当前主题样式,同 `addSettingButton` 克隆导航条目的既有套路)。
- IDE 底部的原生输入/输出区外包一层 **TAB 容器**:`自定义测试 | 样例测试` 两个标签页。
  - 「自定义测试」页 = 原生输入/输出框,DOM 原样保留,只做显隐;
  - 「样例测试」页 = 本功能的样例结果面板;
  - 两个标签随时可手动互切,互不破坏。
- 点「一键测试样例」自动切到「样例测试」标签并开始测试。

### 2.2 样例行

- 每组样例一行。行头 = 折叠箭头 + `样例 #N` + 耗时/内存(如 `12ms · 848KB`) + 状态胶囊。
- 手风琴:同一时刻最多展开一行;点行头切换展开/折叠。
- 运行中自动展开当前正在测试的行;全部结束后自动定格在**首个未通过**的行(全部通过则全部折叠)。
- 展开区三栏并列:`输入 | 期望输出 | 实际输出`,等宽字体。
  - 期望/实际中不一致的行做**逐行 diff 红色高亮**;
  - 每栏配一个小复制按钮(复用脚本现有复制基建);
  - 面板窄时三栏降级为上下堆叠。
- CE(编译错误)时:所有行标 CE,展开区不显示三栏,改为显示接口返回的编译错误日志。
- TLE 等无输出场景:实际输出栏显示占位说明(如「超时,未产生输出」)。

### 2.3 状态胶囊

- 全集:`等待 / 运行中 / AC / WA / TLE / MLE / RE / OLE / CE / UKE`。
- 配色与洛谷原生评测状态配色**完全一致**——实现期从洛谷记录页真实 CSS 取色校准,不自造色值。
- 自测场景实际出现的状态是全集子集;接口返回无法映射的状态一律归 UKE。

### 2.4 面板头部

- 标题「样例测试」+ 汇总条(`测试中… / N/M 通过 · x WA · y TLE / 已停止`)。
- 「重新测试」按钮;运行中显示「停止」按钮(当前组跑完即停,剩余标「未测」)。
- **结果过期标注**(owner 已批):监听编辑器内容变更,测试完成后代码再被修改时,汇总条追加「结果可能已过期,建议重新测试」。

### 2.5 运行策略

- 逐组**串行**测试;组间加约 500ms 最小间隔防限流;收到限流响应(如 429)等待后重试一次。
- 某组失败(WA/RE/TLE 等)**继续跑完**全部样例;CE 例外——直接全部标 CE 并停止。
- 每组设超时护栏 30s(接口逾期未返回则该组标 UKE、耗时位注「未返回」,继续下一组)。
- 运行中禁用「一键测试样例」按钮防重入;用户此时手动触发原生「自测」会互相干扰,给出提示。

### 2.6 边界情形

- **自定义输入保护**:驱动原生流程会改写原生输入框——开始前快照用户内容,全部结束后还原。
- **无样例题目**(交互题/提交答案题等):按钮置灰并提示「本题无样例」。
- 语言选择、O2 等编译选项:天然继承原生自测流程当前设定,不另行处理。
- 代码为空:直接提示,不发请求。

## 3. 技术方案

### 3.1 样例来源

- 首选 `window._feInstance` 内的题目数据 `samples` 数组(洛谷页面自身渲染样例所用的同一份数据;脚本已有 `harvestFromFeInstance` 先例)。
- SPA 换页兜底:同源 `?_contentOnly=1` 接口获取。
- 禁止从题面 DOM 爬样例文本。

### 3.2 测试引擎:驱动原生流程 + 网络拦截

- 逐组把样例输入写入原生输入框 → 程序化触发原生「自测」;
- 结果**不扒 DOM**:包一层 `fetch`/XHR 拦截,捕获自测接口的响应 JSON,读取输出、评测状态、用时、内存(响应含内存字段则显示,不含则该项隐藏)。
- 具体拦截点(接口 URL、请求/响应字段)在实现期用登录态浏览器观察一次真实流量后钉死;该侦察是实现的第一步。

### 3.3 判定口径

- 与洛谷官方一致:CRLF 归一化为 LF,忽略每行行尾空格,忽略末尾多余空行/换行。
- AC/WA 由脚本按上述口径比较捕获输出与期望输出得出;RE/TLE/CE 等从接口响应状态字段映射。

### 3.4 工程约束(对齐 AGENTS.md)

- 功能开关登记 `FEATURE_LABELS` + 底部 `FEATURES`,缺一不可。
- IDE 模式相关的所有新页面选择器集中进 `SELECTORS`。
- 仅同源请求;不扩大 `@match`;不新增第三方依赖与权限。
- SPA 兼容:按钮/tab 容器由 MutationObserver + history 路由监听模式补挂;换题/进出 IDE 模式时清理面板与运行状态,防重复挂载与重复绑定。
- 样式:按钮/标签克隆原生控件;自绘部分(样例行、三栏)贴洛谷灰阶与圆角规范;状态胶囊用记录页真实色值。

## 4. 组件划分

| 单元 | 职责 | 依赖 |
| --- | --- | --- |
| `sampleSource` | 取当前题目样例数组(_feInstance 优先,_contentOnly 兜底) | 洛谷同源数据 |
| `selfTestDriver` | 单组样例:写输入 → 触发原生自测 → 经拦截层拿响应;超时护栏 | SELECTORS、拦截层 |
| `selfTestInterceptor` | fetch/XHR 包装,识别自测接口调用并把响应交给等待中的 driver | 无(启动时安装一次) |
| `verdictJudge` | 归一化比较输出,产出 AC/WA;映射接口状态到胶囊全集 | 无 |
| `samplePanel` | tab 容器、样例行列表、手风琴、diff 渲染、汇总条、停止/重测/过期标注 | verdictJudge 结果 |
| `batchRunner` | 串行调度、间隔与重试、CE 短路、输入快照与还原、防重入 | 上述全部 |

## 5. 错误处理

- 拦截不到自测响应(洛谷改接口):该组标 UKE、耗时位注「未返回」,面板给出「脚本可能需要更新」提示,不静默。
- 样例数据取不到:按钮置灰提示,不发请求。
- 任何异常走 `console.error("LuoguSP …")` 既有日志约定,不阻断洛谷原生功能。

## 6. 验证清单(实现后)

- 多样例题(≥3 组)、单样例题、无样例题(交互/提答)。
- WA diff 高亮、TLE 无输出占位、CE 编译日志展开。
- 自定义输入内容在批测后完整还原。
- SPA:题目间跳转、进出 IDE 模式,按钮/tab 不重复挂载、状态不残留。
- 设置面板开关本功能生效;其余功能不受影响。
- 发布元数据:`@version`、README 徽记同步。

## 7. 非目标

- 不做多组样例并行测试(串行是有意选择,防限流)。
- 不做自造样式的独立浮层 UI(已被 TAB 方案取代)。
- 不做对拍/自定义数据生成等超出「跑题面样例」范围的能力。
